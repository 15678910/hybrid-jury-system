# 시민법정 커뮤니티 운영 가이드

## 개요

이 가이드는 시민법정 커뮤니티(카카오톡 오픈채팅방, 텔레그램)에서 민주적 의사결정을 진행하는 방법을 설명합니다.

참여자들이 직접 제안하고, 투표하고, 운영 규칙을 스스로 만들어가는 것이 목표입니다.

**커뮤니티 규칙**: https://communityrule.info/create/?r=1767161408702

---

## 채널별 용도 구분

| 채널 | 용도 | 특징 |
|------|------|------|
| **웹사이트** | 심도 있는 정책 논의 | 장기 투표(1개월), 10명 추천 시 자동 승격, 상세한 설명 가능 |
| **텔레그램/카카오톡** | 간단한 설문 및 빠른 의견 수렴 | 단기 투표(24시간~7일), 즉시 투표 생성 |

**언제 어디서 제안할까요?**

- **웹사이트**: 커뮤니티 규칙 변경, 중요 정책 결정, 충분한 토론이 필요한 사안
- **텔레그램/카카오톡**: 일정 조율, 간단한 의견 수렴, 빠른 결정이 필요한 사안

---

## 1. 웹사이트 주제 제안하기

시민법정 웹사이트(https://시민법정.kr)에서 직접 새로운 투표 주제를 제안할 수 있습니다.

### 1.1 제안 방법

1. 메인 페이지에서 **"+ 주제 제안하기"** 버튼 클릭
2. 제안할 주제와 설명 작성
3. 제출 후 다른 시민들의 추천을 받습니다

### 1.2 자동 승격 조건

**추천 기간 내 추천 수 10명 이상**이면 자동으로 "진행 중인 투표" 섹션으로 이동되어 정식 투표가 시작됩니다!

### 1.3 처리 흐름

1. **제안 등록**: 제안방에 표시됨
2. **시민 추천**: 다른 시민들이 👍 버튼으로 추천
3. **자동 승격**: 10명 이상 추천 시 → 정식 투표로 자동 승격!

## 2. 카카오톡 오픈채팅방 운영 방법

카카오톡은 봇 자동화가 불가능하므로 **수동**으로 운영합니다.

### 2.1 제안하기

누구나 아래 형식으로 제안할 수 있습니다:

```
#제안 [제안 내용]
```

**예시:**
```
#제안 월례회의를 매주 토요일 오후 3시로 변경하자
#제안 신규 회원 환영 메시지를 추가하자
#제안 커뮤니티 규칙에 '상호 존중' 조항을 추가하자
```

### 2.2 투표 만들기 (방장/관리자)

제안이 올라오면 방장이 투표를 생성합니다:

1. **채팅방 메뉴** (우상단 ≡) 클릭
2. **투표** 선택
3. 투표 내용 입력:
   - **제목**: `[제안] 제안 내용`
   - **선택지**:
     - 찬성
     - 반대
     - 기권
   - **마감일**: 24시간 후 (또는 적절한 기간)
   - **복수 선택**: 끄기
   - **익명 투표**: 끄기 (공개 투표 권장)
4. **완료** 클릭

### 2.3 투표 결과 공지 (방장/관리자)

투표가 마감되면 결과를 공지합니다:

```
📊 투표 결과 발표

📝 제안: [제안 내용]
👤 제안자: [이름]

✅ 결과: 통과 / 부결

📈 투표 현황:
  찬성: OO표
  반대: OO표
  기권: OO표
  총 참여: OO명

[통과 시] 커뮤니티 규칙에 반영됩니다.
```

### 2.4 규칙 반영

통과된 제안은 CommunityRule에 반영합니다:
1. https://communityrule.info/create/?r=1767161408702 접속
2. 해당 규칙 수정/추가
3. 저장 후 링크 공유

## 3. 텔레그램 운영 방법 (자동화)

텔레그램은 봇이 자동으로 처리합니다.

### 3.1 제안하기

그룹에서 아래 형식으로 메시지를 보내면 **자동으로 투표가 생성**됩니다:

```
#제안 [제안 내용]
```

**예시:**
```
#제안 월례회의를 매주 토요일 오후 3시로 변경하자
```

### 3.2 투표 기간 조정 (선택사항)

제안자가 직접 투표 기간을 설정할 수 있습니다:

```
#제안 48시간 [제안 내용]
#제안 7일 [제안 내용]
```

**기간 설정 규칙:**
- **기본값**: 24시간 (기간 미지정 시)
- **시간 단위**: 1~240시간 (예: `#제안 48시간 내용`)
- **일 단위**: 1~10일 (예: `#제안 3일 내용`)

**예시:**
```
#제안 48시간 긴급 안건에 대해 투표합니다
#제안 7일 중요한 규칙 변경 제안입니다
```

### 3.3 설문하기 (#설문 태그)

간단한 의견 수렴이나 일정 조율에는 `#설문` 태그를 사용합니다:

```
#설문 [질문 내용]
```

**예시:**
```
#설문 다음 정기모임 날짜는 언제가 좋을까요?
#설문 이번 주 토요일 오프라인 참석 가능하신가요?
#설문 48시간 새 로고 디자인 A안과 B안 중 어떤 게 좋나요?
```

### 3.4 다중 선택 투표하기 (#투표 태그)

일정 조율 등 여러 선택지 중 하나를 고르는 투표에는 `#투표` 태그를 사용합니다:

```
#투표 질문 내용
- 선택지1
- 선택지2
- 선택지3
```

**예시:**
```
#투표 오프라인 모임 날짜 선택
- 1월 2일
- 1월 5일
- 1월 9일
```

```
#투표 48시간 다음 프로젝트 주제 선택
- AI 챗봇 개발
- 웹사이트 리뉴얼
- 모바일 앱 개발
```

**주의사항:**
- 선택지는 최소 2개, 최대 10개까지 가능
- 각 선택지는 `-` 또는 `•`로 시작
- 투표 종료 후 가장 많은 득표를 받은 선택지가 1위로 표시됨

**#제안 vs #설문 vs #투표 차이점:**

| 구분 | #제안 | #설문 | #투표 |
|------|-------|-------|-------|
| 용도 | 커뮤니티 규칙 변경 | 간단한 예/아니오 질문 | 다중 선택지 투표 |
| 선택지 | 찬성/반대/기권 | 예/아니오/잘 모르겠음 | 직접 지정 (2~10개) |
| 결과 | 통과/부결 판정 | 응답 현황만 표시 | 1위 선택지 표시 |
| 예시 | 규칙 변경 제안 | 참석 가능 여부 | 일정/장소 선택 |

### 3.5 자동 처리 흐름

1. **태그 감지**: 봇이 `#제안`, `#설문`, `#투표`로 시작하는 메시지 감지
2. **알림 전송**: "새로운 제안/설문이 등록되었습니다!" 메시지 전송
3. **투표 생성**: 자동 투표 생성 (지정 기간 또는 기본 24시간)
4. **결과 발표**: 마감 후 자동으로 결과 공지
5. **기록 저장**: 모든 제안/설문과 결과가 데이터베이스에 저장됨

### 3.6 투표 참여

생성된 투표에서 선택지를 클릭하면 됩니다:

**#제안 투표:**
- ✅ 찬성
- ❌ 반대
- ⏸️ 기권

**#설문 투표:**
- 👍 예
- 👎 아니오
- 🤔 잘 모르겠음

**#투표:**
- 제안자가 지정한 선택지 중 선택

## 4. 민주적 의사결정 원칙

### 4.1 투표 규칙

| 항목 | 내용 |
|------|------|
| 투표 기간 | 기본 24시간 (제안자가 1시간~10일 범위 내 조정 가능) |
| 통과 기준 | 찬성 > 반대 (기권 제외) |
| 동률 처리 | 부결 |
| 최소 참여 | 없음 (추후 조정 가능) |

### 4.2 제안 가이드라인

**좋은 제안:**
- 구체적이고 실행 가능한 내용
- 커뮤니티 전체에 영향을 주는 사안
- 명확한 변경 사항 제시

**피해야 할 제안:**
- 개인적인 요청
- 모호하거나 추상적인 내용
- 이미 진행 중인 동일 제안

### 4.3 토론

- 투표 전 충분한 토론을 권장합니다
- 서로 다른 의견을 존중해주세요
- 질문이 있으면 제안자에게 물어보세요

## 5. CommunityRule 활용

CommunityRule은 커뮤니티 규칙을 시각적으로 정리하는 도구입니다.

**현재 규칙 보기**: https://communityrule.info/create/?r=1767161408702

### 5.1 규칙 구성 요소

- **Culture (문화)**: 커뮤니티의 가치와 규범
- **Decision (의사결정)**: 투표 방식, 합의 방법
- **Process (절차)**: 가입, 제명, 분쟁 해결 등
- **Structure (구조)**: 역할, 위원회, 조직 구조

### 5.2 규칙 수정 절차

1. `#제안`으로 규칙 변경 제안
2. 커뮤니티 투표 진행
3. 통과 시 CommunityRule 업데이트
4. 변경된 규칙 공지

## 6. 자주 묻는 질문

### Q: 투표 기간을 변경할 수 있나요?
A: 네, 텔레그램에서는 제안자가 직접 투표 기간을 설정할 수 있습니다. `#제안 48시간 내용` 또는 `#제안 7일 내용` 형식으로 작성하면 됩니다. 기간을 지정하지 않으면 기본 24시간이 적용됩니다. 카카오톡에서는 방장이 투표 기간을 조정할 수 있습니다.

### Q: 내 제안이 부결되면 다시 제안할 수 있나요?
A: 네, 하지만 내용을 수정하거나 보완한 후 다시 제안하는 것을 권장합니다.

### Q: 투표를 취소할 수 있나요?
A: 제안자가 요청하면 방장이 투표를 취소할 수 있습니다 (텔레그램은 자동 진행).

### Q: 익명 투표는 안 되나요?
A: 민주적 투명성을 위해 공개 투표를 기본으로 합니다. 필요 시 제안으로 변경 가능합니다.

### Q: #제안과 #설문의 차이점은 무엇인가요?
A: `#제안`은 커뮤니티 규칙 변경 등 중요한 의사결정에 사용하며, 결과에 따라 통과/부결이 판정됩니다. `#설문`은 일정 조율이나 간단한 의견 수렴에 사용하며, 응답 현황만 보여주는 참고용입니다.

### Q: 웹사이트 제안과 텔레그램 제안 중 어디서 해야 하나요?
A: 중요한 정책이나 규칙 변경은 웹사이트에서, 빠른 의견 수렴이나 일정 조율은 텔레그램/카카오톡에서 하시면 됩니다.
