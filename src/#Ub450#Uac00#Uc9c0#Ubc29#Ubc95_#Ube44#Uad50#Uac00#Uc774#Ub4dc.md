# 🎯 두 가지 자동 팝업 방법 선택 가이드

## 📦 제공되는 파일

1. **App.jsx** - 방법 1: 항상 자동 팝업 (추천!)
2. **App_방법2_첫방문자만.jsx** - 방법 2: 첫 방문자만 팝업

---

## 🔥 방법 1: 항상 자동 팝업 (추천!)

### 작동 방식:
```
누가 언제 접속하든
매번 팝업이 뜹니다!
```

### 장점:
- ✅ **최대 노출**: 모든 방문자가 100% 포스터 확인
- ✅ **강력한 임팩트**: 매번 음악 + 포스터
- ✅ **간단함**: 쿠키/로컬스토리지 불필요
- ✅ **재방문자**: 다시 와도 팝업 (상기 효과)

### 단점:
- ⚠️ 자주 방문하는 사람은 번거로울 수 있음
- ⚠️ 매번 X 버튼 눌러야 함

### 추천 대상:
```
✅ 캠페인 초기 단계
✅ 최대 전환율이 목표
✅ 단기 프로모션
✅ 강렬한 메시지 전달
```

### 사용 사례:
```
- 서명 캠페인 (지금!)
- 긴급 공지
- 이벤트 프로모션
- 신규 사이트 런칭
```

---

## 🎯 방법 2: 첫 방문자만 팝업

### 작동 방식:
```
첫 방문: 팝업 뜸 ✅
재방문: 팝업 안 뜸 ❌
(30일 동안)
```

### 장점:
- ✅ **사용자 친화적**: 재방문자 배려
- ✅ **깔끔한 경험**: 한 번만 봄
- ✅ **재방문 유도**: 번거롭지 않음

### 단점:
- ⚠️ 재방문자는 포스터 못 봄
- ⚠️ 쿠키 삭제하면 다시 뜸
- ⚠️ 시크릿 모드에서 매번 뜸

### 추천 대상:
```
✅ 장기 운영 사이트
✅ 재방문자가 많을 때
✅ 사용자 경험 중시
✅ 일반 기업 사이트
```

### 사용 사례:
```
- 장기 캠페인
- 회원 사이트
- 뉴스레터 구독
- 정보 제공 사이트
```
